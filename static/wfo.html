Reference

  • Hex commands. Each pair of hex digits is one draw step on the 8×8 grid (e.g. 4F). A byte is a xxx b yyy: xxx and yyy are steps along X and Y (0–4 forward; 5–7 wrap to −1, −2, −3). The bits ab choose the stroke—00 line, 01 or 10 arc (quarter when both axes move, half when only one), 11 move without drawing. If there is no movement: 00 snap to row top, 01 dot, 10 circle, 11 no-op.
  • Path notation. Write moves as arrows; add a sans-serif bold digit only for 2+ steps (← = 1 left, →𝟮 = 2 right, ↓𝟰 = 4 down). Adjacent arrows are one compound move (←↑ = diagonal). Separate strokes with ⋅ (←⋅↑ = left then up). Invisible moves: ⮜ ⮞ ⮝ ⮟ (same counting rule). Arcs: ◖ horizontal bow, ◗ vertical bow. Also ⯭ snap, ∗ dot, ⍛ minicircle. Spelling out 1 (→𝟭) still works.
  • Layout. ⌇⌇ starts a new row; ⌇ separates glyphs on the same line. In the editor, ⌈ marks glyph start (cyan) and ⌇ marks glyph end (magenta). Lines beginning with // are comments.
  • Replacements. Rule lines like 13,24 map a source byte to a target. Pairs on one line apply together; later lines run in order. Use \, for a literal comma in the source.
  • Includes. {use_rule set.order} and {use_input set.order} pull in another card's rule or input. Nested tokens expand; cycles and missing cards log a warning and leave the token empty.
  • SVG background. Enter {use_svg name.svg}, a filename, pasted SVG markup, or a URL. Files in public/svg load from /svg/.
