Digit Pair Instructions

  • Digit pairs drive drawing. Treat every two hexadecimal characters (for example 4F) as a single command on the 8×8 grid.
  • Bit layout matters. The byte splits into a xxx b yyy; the three-bit values xxx/yyy set horizontal/vertical offsets. Values 0–4 move forward, 5–7 wrap to −1, −2, −3.
  • Command bits pick the stroke. The outer bits a and b (written together as ab) choose the action: 00 draws a straight line, 01 and 10 draw quarter arcs when both axes move or half arcs when only one axis moves (vertical or horizontal bow), and 11 moves without drawing.
  • Zero moves are special. With no movement, 00 snaps to the row top, 01 plots a point, 10 draws a full circle, and 11 does nothing.
  • Path notation in rules. Visible lines use arrows with optional count (default 1), using sans-serif bold digits 𝟬–𝟵 so counts never clash with hex: ←𝟮 left, →𝟯 right, ↑𝟭 up, ↓𝟰 down (counts 𝟭–𝟵 supported). Chain segments with ⎹ (e.g. ←𝟮⎹↓𝟰). Diagonal strokes combine axes in one token (e.g. ←𝟮↑𝟯 or ←𝟵↑𝟵). Invisible moves use ⮜𝟮, ⮞𝟯, ⮝𝟭, ⮟𝟰 (e.g. ⮞𝟯⎹⮝𝟭). Arcs: ◖→𝟮↑𝟯 horizontal bow, ◗→𝟮↑𝟯 vertical bow. Special: ⯭ snap, ∗ dot, ⍛ minicircle. Card 1.4 holds the shared path→hex vocabulary; glyph cards usually end with {use_rule 1.4}.
  • Layout helpers keep things tidy. Use ⌇⌇ (two ⌇) to start a new word/row, a single ⌇ to delimit successive glyphs, and prefix lines with // for comments that the renderer ignores. Literal hyphens in text stay as -.
  • Rules transform the raw data. Add replacements like 13,24 (source,target) to the Rule box; pairs on the same line apply simultaneously, then later lines cascade before glyphs are drawn. Use \, for a literal comma source.
  • Pull in another card's rule or input. Tokens {use_rule 1.2} and {use_input 1.2} can appear anywhere in the Rule or Input box; each is replaced by that card's field, and nested tokens in the inserted text are expanded too. A cycle or missing card logs a warning and inserts nothing for that token.
  • SVG backgrounds from public/svg. In the SVG field use {use_svg moon.svg} or just moon.svg (files are served at /svg/). You can still paste full SVG markup or use a remote URL.
